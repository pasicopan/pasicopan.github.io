原文：On the fly WebP decoding using WASM and a Service Worker
简单翻译，不喜勿喷

译者前言

（原作者太牢骚，我先直接把他的成果github 地址放出来。github）

找不到原作者的online demo ，我给大家复制一份online demo:

前言

我将要给大家展示的一个小demo，是一个已经成功在chrome 和firefox 里面解码webp格式图片的demo。
（原因是）libwebp 已经基于SDL 有了一些 WASM 实验性支持，这是emscripten已经在 canvas 元素上所支持的。

（只是基于WASM的解码工具）虽然已经工作得挺好了，但人们需要额外引入一些library 似乎还不够友好。通常还需要手动调用JavaScript function 去解码，如果想在css 里面使用就更难操作了。

我非常兴奋地发现，service worker 里面可以使用 WebAssembly。（这样即使在css 里面也可以愉快地自动转码了）

（略过作者一些牢骚和失败的尝试）

接下来就简单了，我要做的只是暴露两个方法出来就好，分别是decode()和 getInfo()，后者（getInfo）是非常有必要的，因为我需要知道图片的体积才能够正确地初始buffers。巴拉巴拉。。。

现在我们已经有个library 了，我们还需要封装一下它，使之更好用，不需要人们去知道buffer，allocations 等麻烦的东西。

接下来我们创建一个mini 的js library 叫做webp-decoder.js。这是一个可以在主线程和worker 线程都能使用的library。将会暴露出一个异步函数叫做fetchWebPDecoder 并返回一个WebPDcoder 对象和一些其他方法。

然后是开始写service worker的注册

然后是webp-decoder.js 的加载 wasm.js 的方法

（重点知识终于来了，同时我深感自己翻译得很不好）
webpdecoder 构造器需要一个ArrayBuffer 作为参数，我们将为此创建一个带类型的array，一个Uint8Array 作为libwebp 需要的const uint8_t* 指针。
我们需要把数据复制到WebAssembly的堆里面。具体做法就是使用Emscripten 提供的两个api _malloc 和Module.HEAPU8.set()，根据byteLength 去使用_malloc 申请需要的空间，然后使用Module.HEAPU8.set() 把数据复制过去。

使用cwrap，我很容易就能从C 里面拿到想要的数据，例如version, 图片尺寸，宽高等

在js 方面，我们根据Emscripten getInfo 的结果，再一次初始化足够的内存空间

C 的内部也会做一些分配（内存）的操作，并返回uint8_t的buffer 值，然后我们需要转成Uint8ClampedArray 给js 使用。因为ImageData 只能用Clamped array，所以我们只能转成Uint8ClampedArray。

解码后的数据并不是我们想要的，我们最后想要的其实是一个格式化为image/png 的blob 数据。我们还有一些小工作要做。我们创建一个的canvas，并使用相关api 进行转换。

好了！我们已经写好一个library 去解码 webp 图片！

怎么用

看图吧，加载个图片，然后转为buffer，初始化解码器，使用canvas 转为blob，赋值给img 标签

service worker 里面也类似，看图，注册好service worker，判断请求是不是一个webp 的GET 请求，是就拦截并另外进行下载和转码
